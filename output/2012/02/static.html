STATIC WEBSITES

Hacking like a blogger

There's no shortage of static website generators these days. After years of storing every single bit in databases, we apparently came to the conclusion that, when it comes to personal sites and blogs, filesystems aren't that bad after all. Of course this is nothing new, initially the web consisted of static files, but in the first part of the last decade every tiny little blog ran a database.

I always found the static approach charming, I think it first caught my attention when I first learned Ruby and encountered Hobix 8 years ago or so. Later I was fascinated by an article written by Martin Fowler, in which he describes how he build his site with Rake. Also I was aware that ongoing was a static site.

I think a relevant catalyst for the current boom of static websites has been a blog post by Tom Preston-Werner: Blogging like a hacker. Another reason for this trend might be that writing a tool to generate your personal website is a nice little programming project and that's exactly the reason why I wrote one too.

Today static website generators come in all flavors: Jekyll, Octopress, Mynt, Middleman, Stasis, Nesta. Even Marco wrote one.

Blake

To build this website I wrote Blake; a small Node.js module, that provides a simple infrastructure to generate static sites. Blake leaves the actual transformation from input to output to you, which makes it rather flexible. Blake takes advantage of Node's non-blocking IO capabilities and runs its tasks in parallel. It makes minimal assumptions and doesn't limit your choices of markup language and template engine. To be honest, it doesn't do terribly much. Blake drives the generation process and stays out of the way. It can be used from the command-line or as library.

Install

npm install -g blake

Command-line usage

blake input output [input/file …]

Deployment

Of course you could build your site locally and upload it to your webserver manually, but obviously the recommended approach is to run Blake on your server and use post-receive hooks to automatically generate your site on your server everytime you're pushing to your input data repository.
